1.组长或项目经理，本地初始化版本库，本地可以创建多个仓库
进入某个文件夹，打开git
git init
有.git 文件，说明仓库创建好了



2.创建当前项目的共有的文件和目录
css  reset.css
js	jquery.js
imgs sprite.png
index.html

3.组长要把文件添加到暂存区
git add .

4.添加到本地仓库
git commit -m  "描述信息"

如果修改了内容，需要提交到本地仓库，形成版本
如果想要回退到之前的版本，需要安装一个插件，git history，
重新加载后，编辑器的右上角会出现git的图标，点击后进入历史记录的界面

只要commit，就会形成版本
想要在各个版本之间切换，需要使用git checkout 版本号  命令，或者在vscode中打开git history界面会罗列出所有的版本，点击想要切换到的版本的右边的第二个图标，在弹出的窗口中选择checkout命令即可

5.组长本地的文件目录确定好之后，需要上传文件的git服务器（github-国外的，速度慢，代码开源公开  或 码云-国内的，速度快，可以有私有仓库）

1>登陆码云
2>创建远程仓库
3>配置远程和本地的连接，git remote add origin “远程仓库的地址”
在vscode中点击右上角的拉取，输入用户名和密码即可
4>第一次运行命令 git pull --rebase origin master
5>将共有的文件上传到服务器，git push -u origin master
到此为止，组长已经把文件上传到服务器上

组员：
1.打开文件夹，运行 git clone ”远程仓库地址“
2.添加功能，点击push提交到远程服务器

组员需要克隆组长的项目，需要组长先发邀请
组长：进入当前仓库，选择右上角的管理，选择仓库成员管理，可以直接输入组员的用户名添加，也可以链接邀请
这样组员同意后就可以在本地操作组长创建的服务器端的项目了

五个同学共同完成一个项目：
1.组长邀请成员   进入仓库--》管理--》仓库成员管理--》开发者--》添加开发者--》链接邀请成员

组员登陆码云，右上角消息中同意即可管理同一个项目
